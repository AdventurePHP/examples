Class loading

As of version 2.0 classes, templates, and configuration files can be loaded from different base paths. This allows to separate framework and application code as well as productive code from test artifacts.

For this reason, the APF provides a ClassLoader mechanism based on PSR-0 to load application content from different base folders. Configuration can be one within one bootstrap file by registering any number of ClassLoaders.
